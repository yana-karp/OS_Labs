### 1. Порівняйте файлові структури Windows-подібної та Linux-подібної системи.
   
| Критерій              | Windows-подібна система                               | Linux-подібна система                                |
| --------------------- | ----------------------------------------------------- | ---------------------------------------------------- |
| Верхній рівень        | «Мій комп’ютер» / This PC                             | Кореневий каталог `/`                                |
| Диски                 | Кожен диск має окрему літеру (C:, D:)                 | Немає літер; усе підключається в одну структуру      |
| Структура             | Розділена за дисками                                  | Єдина ієрархія                                       |
| Підключення пристроїв | Відображаються як окремі диски                        | Монтуються у директорії (наприклад `/mnt`, `/media`) |
| Шляхи                 | `C:\Users\Name\file.txt`                              | `/home/name/file.txt`                                |
| Системні файли        | Розташовані в різних місцях (наприклад, `C:\Windows`) | Чітко структуровані (`/etc`, `/bin`, `/var` тощо)    |
| Розширення файлів     | Важливі (.exe, .txt)                                  | Не обов’язкові (тип визначається вмістом)            |
| Права доступу         | Спрощена система прав                                 | Розширена система прав (r, w, x)                     |
| Користувачі           | Дані зазвичай у `C:\Users`                            | Дані у `/home`                                       |
| Гнучкість             | Менш гнучка структура                                 | Висока гнучкість і стандартизація (FHS)              |
> **Основні відмінності:**
> 1. Організація дисків
>    * Windows: кожен диск окремо (C:, D:)
>    * Linux: всі диски — частина єдиного дерева /
> 2. Підключення пристроїв
>    * Windows: автоматично як новий диск
>    * Linux: вручну/автоматично монтується в папку
> 3. Стандартизація
>    * Linux має стандарт FHS (чіткі призначення папок)
>    * Windows — менш жорстка структура
> 4. Робота з файлами
>    * Windows залежить від розширень
>    * Linux — від типу вмісту та прав доступу
### 2. Розкрийте поняття FHS. Як даний стандарт використовується в контексті файлових систем? 
**FHS (Filesystem Hierarchy Standard)** — це стандарт ієрархії файлової системи, який визначає загальну структуру директорій та їх призначення в Linux-подібних операційних системах.

Стандарт встановлює єдині назви та призначення директорій верхнього рівня, завдяки чому більшість Linux-дистрибутивів мають однакову структуру файлової системи. Це означає, що користувач або адміністратор, який працював з однією FHS-сумісною системою, без труднощів орієнтуватиметься в іншій.

Відповідно до стандарту, кожна директорія має чітко визначене призначення. Наприклад, **_/bin_** містить основні утиліти користувача, **_/etc_** — файли конфігурації системи, **_/home_** — домашні директорії користувачів, **_/var_** — файли, що часто змінюються (зокрема логи), а **_/tmp_** — тимчасові робочі файли. Такий розподіл забезпечує чіткий порядок між системними, користувацькими та тимчасовими файлами.
### 3. Перерахуйте основні команди для роботи з файлами та каталогами в Linux: створення, переміщення, копіювання, видалення. 
**Створення файлів і каталогів**

Для створення нового порожнього файлу використовується команда _**touch**_. Вона не розміщує жодних даних у файлі — його розмір становить 0 байт, однак сам факт існування файлу може впливати на роботу деяких команд або сервісів. Для створення нової директорії використовується команда _**mkdir**_ (make directory).
___
**Копіювання файлів і каталогів**

Копіювання здійснюється за допомогою команди _**cp**_, яка потребує вказання джерела (source) та призначення (destination). Якщо призначенням є директорія, новий файл зберігає оригінальну назву. Щоб надати копії іншу назву, її слід вказати у шляху призначення. За замовчуванням команда cp не копіює директорії — для цього необхідно застосувати параметр **_-r_** (recursive). Варто зважати на те, що якщо файл із такою назвою вже існує в місці призначення, він буде перезаписаний без попередження. Параметр **_-i_** дозволяє уникнути цього, запитуючи підтвердження, а параметр **_-v_** виводить детальну інформацію про виконану операцію.
___
**Переміщення та перейменування файлів**

Для переміщення файлів використовується команда mv. Вона працює схожим чином до cp, однак оригінальний файл при цьому видаляється з початкового розташування. Якщо як призначення вказано директорію — файл переміщується до неї зі збереженням назви. Якщо директорію не вказано, а лише нову назву файлу — файл перейменовується і залишається в тій самій директорії. Команда також підтримує параметри **_-i_** (запит підтвердження перед перезаписом), **_-n_** (заборона перезапису існуючого файлу) та **_-v_** (детальний вивід результату).
___
**Видалення файлів і каталогів**
Для видалення файлів використовується команда _**rm**_. Файли видаляються без будь-якого запиту підтвердження, тому при роботі з кількома файлами одночасно, особливо з використанням символів підстановки (glob), рекомендується застосовувати параметр _**-i**_. Для видалення директорій разом із їх вмістом застосовується параметр _**-r**_ (recursive). Альтернативно, порожню директорію можна видалити командою _**rmdir**_, однак вона не працює, якщо директорія містить файли.
